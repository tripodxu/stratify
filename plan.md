# Stratify 模拟企划书

## 项目目标

通过调节 Stratify 系统中的各项参数，运行大规模随机模拟，观察是否能自然涌现以下典型社会结构：

| 目标结构 | 核心特征 |
|---------|---------|
| 纺锤型社会 | 中间阶层庞大，上下两端窄小 |
| 金字塔社会 | 底层庞大，逐层急剧收窄，顶端极少数 |
| 沙漏型社会 | 中间阶层塌陷，贫富两极分化 |
| 橄榄型社会 | 中产阶级为主体，流动性适中，稳定繁荣 |
| 种姓/封建社会 | 阶层固化严重，代际流动性接近零 |
| 赛博朋克社会 | 极少数顶层垄断资源，底层巨大且完全无上升通道 |

---

## 模拟分阶段计划

---

### 阶段零：基础设施搭建

**目标：** 搭建可运行的模拟框架，产出可视化数据。

**具体操作：**

1. 实现基础数据结构
   - Agent：class, Value, ge, hard, life, maxClass（历史最高阶层）, age, alive
   - World：阶层列表，每个阶层包含 env 系数、Value 需求线、decay 系数
2. 实现时间循环（tick）：每 tick 依次执行积累→需求检查→跌落判定→死亡判定→出生补充
3. 实现随机分布生成器
   - ge：正态分布 N(μ_ge, σ_ge)，截断于 [ge_min, ge_max]
   - hard：正态分布 N(μ_hard, σ_hard)，截断于 [hard_min, hard_max]
   - 初始 life：正态分布 N(L₀, σ_life)，截断于最小寿命
   - 初始 class：可指定分布（均匀 / 金字塔形 / 自定义比例）
4. 输出每 tick 的统计快照：各阶层人数、平均 Value、Value 基尼系数、阶层流动率
5. 可视化：阶层分布堆叠面积图 + 基尼系数折线图 + 流动率折线图

**产出：** 一个可跑通的最小模拟程序，1000 人 × 1000 tick，能看到基本数据曲线。

---

### 阶段一：纯积累实验（无竞争、无寿命）

**目标：** 隔离观察「天赋 + 努力 + 阶层系数」三因素对 Value 分布的影响。

**参数设置：**

| 参数 | 值/范围 | 说明 |
|------|--------|------|
| 总人口 | 1000 | 固定，不生不死 |
| 阶层数 | 5 | 低→高：0, 1, 2, 3, 4 |
| env 系数 | [1.0, 1.5, 2.5, 4.0, 6.0] | 阶层越高系数越大 |
| ge 分布 | N(5, 2)，截断 [1, 10] | 天赋正态分布 |
| hard 分布 | N(5, 2)，截断 [1, 10] | 努力正态分布 |
| 初始 class | 均匀分布（每层 200 人） | 观察自然分化 |
| 积累公式 | speed = env × (ge + hard) / 2 | 算术平均 |
| cross 值 | 暂不启用 | 不可跨层 |
| 竞争 | 暂不启用 | — |
| 寿命 | 暂不启用 | — |

**观察指标：**
- 各阶层 Value 的均值与方差随时间的变化
- 相同 ge/hard 的人在不同阶层的 Value 差距（阶层溢价）
- 1000 tick 后的 Value 分布直方图

**预期发现：**
- 即使初始均匀分布，阶层系数的乘数效应也会迅速拉开差距
- 高阶层的人即使天赋平庸，仅靠 env 系数就能积累远超低层天才的 Value
- 这是「出身决定论」的最简模型

---

### 阶段二：加入跨层机制（有 cross，无竞争）

**目标：** 观察「向上跨越成本」对阶层流动性的影响。

**在阶段一基础上增加：**

| 参数 | 值/范围 | 说明 |
|------|--------|------|
| cross 值 | 方案 A：固定 500；方案 B：随阶层递增 [200, 400, 800, 1600] | 跨越成本 |
| 重返减免 | 历史最高阶层以下享有 50% 减免 | 鼓励曾经的高阶层者重返 |
| 减免衰减 | 每靠近历史最高层一档，减免减少 15% | 阶梯型衰减 |
| 阶级需求线 | 各层 Value 需求 = ge=1, hard=1 的人在该层积累 N tick 的 Value | 维持阶层的最低门槛 |
| 时间缓冲系数 | 前 50 tick 为 0%，50~200 tick 线性增至 100% | 新晋者宽限期 |

**操作：**
1. 个体 Value 达到 cross 值时自动晋升（或可选手动触发）
2. 个体 Value 低于需求线 × 百分比阈值（如 60%）持续 100 tick 时强制跌落
3. 对比两组 cross 方案的结果差异

**观察指标：**
- 跨层事件的频率与方向（向上 vs 向下）
- 各阶层人口比例随时间的变化
- 「中产阶层」（中间层）是否出现膨胀或塌陷

**预期发现：**
- cross 值过低 → 社会流动性过强，阶层失去意义
- cross 值过高 → 阶层固化，底层永远无法上升
- 存在一个「临界 cross 值」使社会结构开始稳定

---

### 阶段三：加入同层竞争

**目标：** 观察同层个体间的资源争夺如何重塑 Value 分布。

**在阶段二基础上增加：**

| 参数 | 值/范围 | 说明 |
|------|--------|------|
| 竞争发起概率 | 每 tick 每人 5% 概率发起竞争 | 控制竞争频率 |
| 竞争对手选择 | 随机选择同层另一人 | 同层竞争 |
| 胜负公式 | score = Value × (0.6 + 0.2×ge/ge_max + 0.2×hard/hard_max) | Value 为主，天赋和努力为辅 |
| 资源转移比例 | 胜者获得败者 Value 的 15% | — |
| 系统损耗 | 败者实际失去 Value 的 20%（多出 5% 消散） | 损耗机制 |
| 跌落触发 | Value 低于需求线 60% 持续 100 tick | 被动跌落 |

**操作：**
1. 每 tick 遍历所有 alive agent，以 5% 概率决定是否发起竞争
2. 随机选取同层对手，按胜负公式结算
3. 记录竞争日志：谁赢了谁，转移了多少 Value

**观察指标：**
- 同层内 Value 的基尼系数（衡量同层不平等）
- 频繁竞争者的阶层变动轨迹 vs 不竞争者
- 是否出现「同层寡头」：少数人垄断同层大部分 Value

**预期发现：**
- 竞争加速 Value 集中，同层内部开始出现分化
- 低天赋高努力的人可能通过频繁竞争弥补天赋不足
- 「马太效应」雏形：Value 越高的人越容易赢得竞争，进一步拉开差距

---

### 阶段四：加入跨层竞争

**目标：** 观察跨层掠夺机制对整体社会结构的影响。

**在阶段三基础上增加：**

| 参数 | 值/范围 | 说明 |
|------|--------|------|
| 跨层竞争概率 | 每 tick 每人额外 2% 概率发起跨层竞争 | 比同层竞争更少见 |
| 目标选择 | 随机选择相邻阶层（上一层或下一层）一人 | 只打相邻层 |
| 发起者系数衰减 | 向上：score × 0.7；向下：score × 0.85 | 跨层成本 |
| 被竞争者加成 | 被跨层挑战时 score × 1.15 | 守方优势 |
| 收获调节 | 高打低：胜者收获 × 0.7；低打高：胜者收获 × 1.3 | 向上突破奖励 |
| 系统损耗 | 同阶段三（败者多失去 5%） | — |

**操作：**
1. 在同层竞争判定之后，额外进行跨层竞争判定
2. 分别记录向上竞争和向下竞争的胜率、平均收获
3. 运行 3 组参数对比：保守（衰减强）/ 中性 / 激进（衰减弱）

**观察指标：**
- 向上竞争的胜率与平均收获（低打高的成功率）
- 向下竞争的频率（高阶层是否频繁掠夺低层）
- 阶层间 Value 流动的净方向（向上流 vs 向下流）

**预期发现：**
- 如果向上突破奖励足够大，会出现「寒门逆袭」少数案例
- 如果向下掠夺过于容易，顶层会持续吸血底层，加速两极分化
- 跨层竞争是阶层流动的核心驱动力，也是社会不平等的放大器

### 阶段五：加入寿命与生死循环

**目标：** 引入代际更替，观察长期社会结构的稳态。

**在阶段四基础上增加：**

| 参数 | 值/范围 | 说明 |
|------|--------|------|
| 初始寿命 | N(800, 150)，截断 [400, 1200] | tick 单位，均值约 800 tick |
| 阶层 decay 系数 | [2.0, 1.2, 0.8, 0.4, 0.1] | 低层损耗最大 |
| 硬努力额外损耗 | hard > 7 时每 tick 额外扣 0.3 | 过劳代价 |
| 死亡处理 | life ≤ 0 时标记 dead，移出模拟 | — |
| 出生补充 | 每 tick 出生人数 = 死亡人数 × 1.0（恒定人口） | 稳态人口 |
| 出生属性 | ge, hard 重新随机，class = 0（底层），Value = 0 | 新生儿从零开始 |
| 代际传承 | 可选：子女 ge = 父母 ge × 0.5 + 随机扰动 | 基因/家教传承 |

**操作：**
1. 每 tick 末尾执行死亡判定与出生补充
2. 追踪「家族谱系」：记录每个 agent 的父母 ID，观察代际阶层变动
3. 分别运行「无传承」和「有传承」两组实验

**观察指标：**
- 社会结构是否在若干代后趋于稳态（各阶层比例稳定）
- 代际流动率：子女阶层与父母阶层的相关系数
- 底层平均寿命 vs 顶层平均寿命的差距（健康不平等）
- 是否出现「阶层遗传」：高阶层子女大概率留在高阶层

**预期发现：**
- 无传承时社会流动性较高，但底层短命导致人口大量更替
- 有传承时逐渐出现阶层固化，「寒门出贵子」概率下降
- 低阶层的高死亡率形成「消耗性底层」，不断有新生儿补充

### 阶段六：参数扫描与结构分类

**目标：** 系统性搜索参数空间，找到能复刻目标社会结构的参数组合。

**操作：**

1. **定义结构分类指标**
   - 基尼系数（Value 分布不平等程度）
   - 阶层熵（人口在各层分布的均匀程度）
   - 代际流动率（子女与父母阶层的相关性）
   - 中间层占比（中间阶层人口占总人口比例）
   - 顶层集中度（顶层 5% 人口持有 Value 的比例）

2. **参数扫描范围**

   | 参数 | 扫描值 | 步长 |
   |------|--------|------|
   | env 增长模式 | 线性 / 指数 / 平方 | 3 档 |
   | cross 值 | 100 ~ 2000 | 200 |
   | ge 分布标准差 | 1 ~ 4 | 0.5 |
   | 竞争频率 | 1% ~ 10% | 1% |
   | 资源转移比例 | 5% ~ 30% | 5% |
   | 阶层 decay 比 | 0.1 ~ 3.0 | 0.5 |
   | 代际传承系数 | 0.0 ~ 0.8 | 0.1 |

3. **扫描执行**
   - 使用网格搜索或拉丁超立方采样，生成 500~1000 组参数组合
   - 每组参数运行 5 次取平均（消除随机性）
   - 每次运行 5000 tick（约 6 代人）
   - 记录稳态后的各项指标

4. **结构聚类**
   - 将各组结果的指标向量做 K-Means 聚类（k=6，对应 6 种目标结构）
   - 检查聚类中心是否与目标结构特征匹配
   - 输出每种结构对应的「典型参数组合」

**产出：**
- 参数空间热力图（各指标随参数变化的趋势）
- 每种目标结构的推荐参数集
- 结构转换边界图（哪些参数变化导致社会结构突变）

### 阶段七：深入分析与论文级产出

**目标：** 从模拟数据中提炼社会学洞察。

**操作：**

1. **敏感性分析**
   - 对每个参数做单变量扫描，固定其余参数，观察各指标的变化率
   - 识别「杠杆参数」：微小变化导致社会结构剧变的参数

2. **相变研究**
   - 重点关注参数空间中的不连续区域（结构突变）
   - 绘制「社会结构相图」：以两个关键参数为轴，标注各区域对应的社会结构

3. **典型社会深度模拟**
   - 从阶段六产出的参数中，选取每种结构的代表参数
   - 运行 20000 tick，人口 10000，详细记录每个个体的完整生命周期
   - 追踪典型案例：底层逆袭者、顶层跌落者、世代中产、世代贫困

4. **涌现现象记录**
   - 是否出现「反直觉」的结果（如努力值过高反而加速死亡）
   - 是否出现周期性波动（阶层比例的振荡）
   - 是否出现「隐性阶级」：名义上同层但 Value 差距巨大

5. **可视化与叙事**
   - 生成每种社会结构的「社会纪录片」：关键个体的生命轨迹动画
   - 制作参数-结构映射的交互式可视化
   - 撰写分析报告，对比模拟结果与现实社会结构的相似性

## 技术实现路线

`
阶段零  →  Python 脚本 + 数据结构 + 基本 tick 循环
阶段一  →  纯积累，无交互，验证公式
阶段二  →  加入 cross 和跌落逻辑
阶段三  →  加入同层竞争
阶段四  →  加入跨层竞争
阶段五  →  加入寿命、死亡、出生、代际
阶段六  →  参数扫描框架（并行化）
阶段七  →  分析与可视化
`

**技术栈建议：**

| 组件 | 选择 | 理由 |
|------|------|------|
| 核心模拟 | Python | 快速迭代，易于调试 |
| 性能热点 | NumPy 向量化 | 万人级模拟需要性能 |
| 参数扫描 | multiprocessing / joblib | 并行跑多组参数 |
| 数据存储 | CSV / Parquet | 每 tick 快照，后续分析用 |
| 可视化 | Matplotlib + Seaborn | 静态图表；Plotly 可选交互图 |
| 分析 | Pandas + SciPy | 统计分析与聚类 |
